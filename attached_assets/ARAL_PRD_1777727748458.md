## 1. PROJECT OVERVIEW

### 1.1 What is ARAL?
ARAL is a full-stack web application that predicts how vulnerable a Filipino college graduate's career is to AI and automation, explains why using SHAP (explainable AI), identifies their skill gaps against CHED curriculum standards, and recommends free online courses to help them upskill.

### 1.2 The Problem It Solves
- 1 in 3 Filipino workers is highly exposed to automation (IMF 2025)
- 800,000+ graduates enter the workforce annually without any personalized automation risk signal
- CHED takes 11 years on average to update curricula (EDCOM II 2026)
- No existing tool gives Filipino graduates a localized, personalized, data-driven career risk assessment

## 3. DATASET: POARD

### 3.1 What is POARD?
**Philippine Occupation-Automation Reference Dataset** — the core dataset that powers the ML model. It is a derived, localized dataset constructed by the research team. It does NOT exist publicly — it is an original research contribution.

### 3.2 POARD Construction (How to Build It)
POARD is built in 4 steps:

#### Step 1: Extract O\*NET Data
- Download O\*NET 29.0 task data from: https://www.onetcenter.org/database.html
- Files needed:
  - `Task Statements.xlsx` — list of all tasks per occupation
  - `Tasks to DWAs.xlsx` — task to detailed work activity mapping
  - `Occupation Data.xlsx` — occupation metadata
  - `Automation Risk Scores` — use Frey & Osborne (2017) automation probability scores mapped to O\*NET SOC codes (available as supplementary data from their paper)
- Target occupations: entry-level roles for the 6 degree programs below

#### Step 2: SOC → PSOC Crosswalk
- Use ISCO-08 as the bridge between US SOC codes and Philippine PSOC 2022 codes
- ILO ISCO-08 crosswalk: https://www.ilo.org/public/english/bureau/stat/isco/isco08/
- PSA PSOC 2022: https://psa.gov.ph/classification/psoc
- Build a mapping table: `SOC_code → ISCO_code → PSOC_code → Occupation_Name_PH`

#### Step 3: Survey Data Integration (1000 Respondents)
- Import cleaned survey CSV (1000 Filipino graduate respondents)
- Each respondent provides:
  - Degree program (categorical)
  - Job title (text → mapped to PSOC)
  - Task distribution (% of workday per O\*NET task category, must sum to 100)
  - Skills checklist (binary Yes/No per CHED CMO competency)
  - Skill proficiency (1–5 Likert per competency)
- Re-weight O\*NET task automation scores using Filipino task prevalence averages per degree program

#### Step 4: Compute Localized Automation Vulnerability Score
For each respondent:
```
vulnerability_score = Σ (task_weight_filipino[i] × automation_probability_onet[i])
```
Where:
- `task_weight_filipino[i]` = respondent's reported % time on task category i / 100
- `automation_probability_onet[i]` = O\*NET automation probability for that task category
- Result: continuous score between 0.0 and 1.0

## 5. SHAP EXPLAINABILITY

### 5.1 Implementation
```python
import shap

explainer = shap.TreeExplainer(model)
shap_values = explainer.shap_values(X_test)

# Global feature importance
shap.summary_plot(shap_values, X_test)

# Individual explanation
shap.force_plot(explainer.expected_value, shap_values[i], X_test.iloc[i])
```

### 5.2 What SHAP Outputs Are Used For

| SHAP Output | Used For |
|---|---|
| Global feature importance ranking | Curriculum gap analysis — which skills matter most across all graduates |
| Per-program SHAP aggregation | Degree-program-specific curriculum recommendations |
| Individual SHAP values | Personal explanation on ARAL results page |
| Negative SHAP contributors | Skills that PROTECT against automation — cross-referenced with CHED CMOs |

### 5.3 Curriculum Gap Analysis Logic
```python
# For each degree program:
# 1. Get top N skills with NEGATIVE SHAP values (protective skills)
# 2. Check if those skills are in the CHED CMO for that program
# 3. If NOT in CMO → curriculum design gap
# 4. If in CMO but low self-report proficiency → curriculum implementation gap

protective_skills = shap_df[shap_df['mean_shap'] < -0.05].sort_values('mean_shap')
ched_skills = get_ched_skills_for_program(degree_program)
missing_from_ched = protective_skills[~protective_skills['skill'].isin(ched_skills)]
```

import google.generativeai as genai

def generate_recommendations(skill_gaps: list, degree_program: str, job_title: str) -> list:
    prompt = f"""
    You are a career advisor for Filipino graduates.
    
    A {degree_program} graduate working as {job_title} has the following skill gaps 
    that make them vulnerable to automation:
    {skill_gaps}
    
    Recommend specific FREE online courses from these platforms only:
    - Coursera (free audit)
    - edX (free audit)
    - Google Digital Garage
    - LinkedIn Learning (free trial)
    - FutureLearn (free)
    - TESDA Online Program (TOP)
    - DepEd Commons
    
    For each gap provide:
    1. Course title
    2. Platform
    3. Direct URL
    4. Why this course addresses the automation risk
    
    Return as JSON array only. No preamble. No markdown.
    """
    
    model = genai.GenerativeModel('gemini-2.5-flash')
    response = model.generate_content(prompt)
    return json.loads(response.text)

## 10. EVALUATION MODULE

### 10.1 System Usability Scale (SUS)
After using ARAL, pilot users are shown a 10-item SUS questionnaire:

```
Rate each statement from 1 (Strongly Disagree) to 5 (Strongly Agree):

1. I think that I would like to use this system frequently.
2. I found the system unnecessarily complex.
3. I thought the system was easy to use.
4. I think that I would need the support of a technical person to use this system.
5. I found the various functions in this system were well integrated.
6. I thought there was too much inconsistency in this system.
7. I would imagine that most people would learn to use this system very quickly.
8. I found the system very cumbersome to use.
9. I felt very confident using the system.
10. I needed to learn a lot of things before I could get going with this system.
```

**SUS Score Calculation:**
```python
def calculate_sus(responses: list) -> float:
    # Odd items (1,3,5,7,9): score - 1
    # Even items (2,4,6,8,10): 5 - score
    odd_sum = sum(responses[i] - 1 for i in [0, 2, 4, 6, 8])
    even_sum = sum(5 - responses[i] for i in [1, 3, 5, 7, 9])
    return (odd_sum + even_sum) * 2.5
    # Score > 68 = acceptable usability
    # Score > 80 = good usability
    # Score > 90 = excellent usability
```
